# Product Manager / Designer Prompts

> **Role focus:** PRD authoring. UI/UX design. User story generation.
> Acceptance criteria definition. Validating PRD against BRD.

## What lives here

| File | Purpose |
|---|---|
| `generate-user-stories.md` | Turn a feature description into 3–5 user stories with acceptance criteria. |
| `validate-prd-against-brd.md` | Check PRD coverage and conflicts against the BRD. |

## Suggested prompts to contribute

As participants work through the training, additional prompts to contribute:

- `expand-acceptance-criteria.md` — turn a thin user story into a detailed AC list.
- `wireframe-from-user-story.md` — turn a user story into wireframe descriptions
  for use with Figma Make.
- `non-functional-requirements-check.md` — check NFRs (performance, accessibility,
  security) are covered for a feature.
- `prd-section-rewrite.md` — rewrite a PRD section that's too vague or too dense.

## How to contribute

1. Create a new `.md` file in this folder with a clear purpose verb.
2. Use the structure: **Role + Context + Task + Format + Constraints**.
3. Wrap variable input in `<<<` and `>>>` delimiters.
4. Test against TaskFlow's existing approved PRD as the quality bar.
5. Submit as a PR. After peer review, it's team property.
