# Solutions Designer Prompts

> **Role focus:** BRD authoring. Stakeholder analysis. Business requirements
> elicitation. Validating BRD content against business objectives.

## What lives here

Reusable prompt templates that Solutions Designers use to draft and refine
Business Requirements Documents (BRDs).

## Suggested prompts to contribute

This folder is intentionally empty as a starting point. As participants work
through the training, they should contribute prompts like:

- `generate-brd-from-stakeholder-notes.md` — turn raw stakeholder input into a
  structured BRD draft.
- `validate-brd-against-business-objectives.md` — check that every BRD
  requirement traces to a business objective.
- `extract-constraints-from-discovery.md` — pull constraints (timeline, budget,
  data residency, etc.) out of discovery notes.
- `interview-questions-for-stakeholder.md` — generate role-specific interview
  questions for stakeholder discovery.

## How to contribute

1. Create a new `.md` file in this folder named with a clear purpose verb
   (`generate-`, `validate-`, `review-`).
2. Use the structure: **Role + Context + Task + Format + Constraints**.
3. Wrap variable input in `<<<` and `>>>` delimiters.
4. Test the prompt against TaskFlow's existing approved BRD as a quality bar.
5. Submit as a PR. After peer review, it's team property.

See `knowledge/prompts/product/generate-user-stories.md` as a reference for
prompt structure and formatting.
