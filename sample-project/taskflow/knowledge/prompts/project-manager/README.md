# Project Manager Prompts

> **Role focus:** Project setup. Sprint coordination. PRD review for
> completeness and clarity. Handoff coordination across phases. Sprint
> tracking, blockers, status communications.

## What lives here

Reusable prompt templates that Project Managers use for project planning,
review, and coordination work.

## Suggested prompts to contribute

This folder is intentionally empty as a starting point. As participants work
through the training, they should contribute prompts like:

- `prd-completeness-review.md` — check a PRD for missing acceptance criteria,
  unclear stories, scope ambiguity.
- `sprint-status-summary.md` — turn raw task status into a stakeholder-facing
  summary.
- `risk-and-blocker-extraction.md` — pull risks and blockers out of standup
  notes or chat threads.
- `handoff-checklist-generator.md` — generate handoff checklists between
  phases (Requirements → Architecture, Architecture → Dev, etc.).
- `stakeholder-update-draft.md` — draft a weekly stakeholder update from
  sprint progress data.

## How to contribute

1. Create a new `.md` file in this folder with a clear purpose verb.
2. Use the structure: **Role + Context + Task + Format + Constraints**.
3. Wrap variable input in `<<<` and `>>>` delimiters.
4. Submit as a PR. After peer review, it's team property.

See `knowledge/prompts/product/validate-prd-against-brd.md` for a structured
review-prompt example.
