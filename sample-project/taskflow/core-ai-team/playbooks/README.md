# Core AI Team — Facilitator Playbooks

> **Internal use only.** This folder is for the Stratpoint core AI team who
> facilitate the AI Foundational Training. Not cohort-facing.

## What lives here

One playbook per module segment, named by week and artifact:

```
core-ai-team/playbooks/
├── README.md                          (this file)
├── role-facilitator-playbook-template.md
├── w1-brd-facilitator-playbook.md     (filled in before W1 Day 1)
├── w1-prd-facilitator-playbook.md
├── w2-architecture-facilitator-playbook.md
├── w2-devtasks-facilitator-playbook.md
├── w3-development-facilitator-playbook.md
└── w3-codereview-facilitator-playbook.md
```

## How this works

Each module's Day 1 (Monday in-person) session is structured as a relay of
role-specialized walkthrough segments. Each segment is led by a core AI team
member who knows that artifact deeply.

| Week | Segment | Facilitator role |
|---|---|---|
| W1 Mon | BRD prompting walkthrough | Core AI team — Solutions Designer specialist |
| W1 Mon | PRD + prototyping prompting walkthrough | Core AI team — Product specialist |
| W2 Mon | Architecture + ADR prompting walkthrough | Core AI team — Solutions Architect specialist |
| W2 Mon | Dev Tasks generation walkthrough | Core AI team — Tech Lead specialist |
| W3 Mon | Implementation prompting walkthrough | Core AI team — Developer specialist |
| W3 Mon | Code review + test prompting walkthrough | Core AI team — Tech Lead specialist |

**Days 2–4 (async):** Each facilitator is on-call consultant for their artifact
via the cohort's communication channel (Slack, Teams, etc.).

**Day 5 (Friday gate review):** Each facilitator co-leads the quality bar review
for their artifact section.

## How to use the template

1. Copy `role-facilitator-playbook-template.md` and rename for your segment.
2. Fill in all sections before Day 1.
3. After the cohort session, update the "Lessons Learned" section.
4. Commit — the playbook improves across cohorts as lessons accumulate.
